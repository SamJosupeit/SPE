Set-SPERegistryKey

Synopsis

Erstellt einen Registry-Key mit dem angegebenen Namen unter dem angegebenen Pfad

Syntax

Set-SPERegistryKey [[-path] <String>] [[-key] <String>] [[-value] <String>] [-Reboot] [-InformationAction <ActionPreference>] [-InformationVariable <String>] [<CommonParameters>]

Description

Erstellt einen Registry-Key mit dem angegebenen Namen unter dem angegebenen Pfad mit dem angegebenen Wert

Notes

Die Cmdlets Test-SPERegistryKey, Remove-SPERegistryKey, Set-SPERegistryKey, Get-SPERegistryKey, Restart-AndRun, Clear-SPERegistryAutostart sowie Set-SPERegistryScriptStatus und Get-SPERegistryScriptStatus werden von den SPE-Standard-Scripts genutzt, um Autostart-Schlüssel in die Windows Registry einzutragen, auszulesen, zu bearbeiten oder zu entfernen.

Examples

Beispiel 1 - Allgemeine Benutzung

PS C:\> $powershell = (Join-Path $env:windir "system32\WindowsPowerShell\v1.0\powershell.exe")
SetRegistry-Key -path "HKLM:\SOFTWARE\Microsoft\Windows\CurrentVersion\Run" -key "Restart-AndResume" -value "$powershell c:\ps1\autostart.ps1"

Erstellt in der Windows Registry unter dem Pfad "HKLM:\SOFTWARE\Microsoft\Windows\CurrentVersion\Run" einen Schlüssel "Restart-AndResume" mit dem Wert "C:\Windows\system32\WindowsPowerShell\v1.0\powershell.exe C:\ps1\autostart.ps1". Das führt dazu, dass das Script "C:\ps1\autostart.ps1" per Autostart beim nächsten Reboot ausgeführt wird.

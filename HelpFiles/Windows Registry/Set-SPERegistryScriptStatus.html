Set-SPERegistryScriptStatus

Synopsis

Trägt für das aktuelle SPE-Standard-Script einen Autostart-Eintrag in die Windows Registry mit Step-Vermerk ein.

Syntax

Set-SPERegistryScriptStatus [[-step] <String>] [-InformationAction <ActionPreference>] [-InformationVariable <String>] [<CommonParameters>]

Description

Trägt für das aktuelle SPE-Standard-Script einen Autostart-Eintrag in die Windows Registry mit Step-Vermerk ein. Diese Funktion ist für Scripts gedacht, die während der Ausführung mehrmals den Computer neu starten müssen, wie dies z.B. bei Installations-Scripts der Fall ist.

Notes

Die Cmdlets Test-SPERegistryKey, Remove-SPERegistryKey, Set-SPERegistryKey, Get-SPERegistryKey, Restart-AndRun, Clear-SPERegistryAutostart sowie Set-SPERegistryScriptStatus und Get-SPERegistryScriptStatus werden von den SPE-Standard-Scripts genutzt, um Autostart-Schlüssel in die Windows Registry einzutragen, auszulesen, zu bearbeiten oder zu entfernen.

Examples

Beispiel 1 - Allgemeine Benutzung

PS C:\> Set-SPERegistryScriptStatus -step "Schritt 1"

In der den SPE-Standard-Scripts zugehörigen Config-Datei Config.ps1 sind die globalen Variablen "$global:RegRunKey"(Wert="HKLM:\SOFTWARE\Microsoft\Windows\CurrentVersion\Run") und "$global:restartKey"(Wert="SPE-Restart") vordefiniert. Diese werden vom Cmdlet Set-RegistryStatus dazu benutzt, um im Autostart-Eintrag der Windows-Registry das aktuelle Script in Verbindung mit einem Step-Vermerk zu hinterlegen. Somit kann das Script nach einem erfolgten Reboot an der entsprechenden Stelle fortgesetzt werden. Dies erfordert allerdings einen entsprechenden Aufbau des Scripts.

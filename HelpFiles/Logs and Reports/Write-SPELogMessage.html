Write-SPELogMessage

Synopsis

Übernimmt eine Text-Message mit weiteren Parametern, um diese in die SPE-Standard-Script-Logdateien zu schreiben.

Syntax

Write-SPELogMessage [[-level] <String> {Critical | High | Medium | Verbose | VerboseEx | Unexpected} ] [[-area] <String> {ListView | ViewField | ListField | Script | WebSite | SiteCollection | List | ListItem | misc | other} ] [[-category] <String> {Added | Removed | Started | Stopped | Aborted | Adding | Removing | Determining} ] [[-CorrelationId] <Guid>] [[-eventId] <String>] [[-process] <String>] [[-thread] <String>] [[-message] <String>] [-InformationAction <ActionPreference>] [-InformationVariable <String>] [<CommonParameters>]

Description

Übernimmt eine Text-Message mit weiteren Parametern, um diese in die SPE-Standard-Script-Logdateien zu schreiben. Für den Einsatz der ULS-Logs werden alle Parameter genutzt, die aber größtenteils mit Default-Werten versehen sind.

Parameters

-level <String>

Gibt die Wichtigkeit der Meldung an. Mögliche Werte sind "Critical", "High", "Medium", "Verbose", "VerboseEx" und "Unexpected". Nur relevant, wenn Logging in ULS aktiviert ist (einzustellen in Config.ps1)

Required? False
Position? 0
Default value VerboseEx
Accept pipeline input? false
Accept wildcard characters? False

-area <String>

Gibt den Bereich, zu dem die Meldung sinnvoll zuzuordnen ist, an. Mögliche Werte sind "ListView", "ViewField", "ListField", "Script", "WebSite", "SiteCollection", "List", "ListItem", "misc" und "other". Nur relevant, wenn Logging in ULS aktiviert ist (einzustellen in Config.ps1)

Required? False
Position? 1
Default value Script
Accept pipeline input? false
Accept wildcard characters? False

-category <String>

Gibt die Category der Meldung an. Mögliche Werte sind "Added", "Removed", "Started", "Stopped", "Aborted", "Adding", "Removing" und "Determining". Nur relevant, wenn Logging in ULS aktiviert ist (einzustellen in Config.ps1)

Required? False
Position? 2
Default value  
Accept pipeline input? false
Accept wildcard characters? False

-CorrelationId <Guid>

Eine Guid, die als Coorelation-ID, ähnlich wie im SharePoint-Umfeld fungiert. Nur relevant, wenn Logging in ULS aktiviert ist (einzustellen in Config.ps1)

Required? False
Position? 3
Default value $global:CorrelationId (in der Config.ps1 definiert)
Accept pipeline input? false
Accept wildcard characters? False

-eventId <String>

Kann als zusätzlicher Parameter besetzt werden, da dieser Wert ebenfalls vom ULS-Viewer interpretiert und gefiltert werden kann. Nur relevant, wenn Logging in ULS aktiviert ist (einzustellen in Config.ps1)

Required? False
Position? 4
Default value 0000
Accept pipeline input? false
Accept wildcard characters? False

-process <String>

Kann als zusätzlicher Parameter besetzt werden, da dieser Wert ebenfalls vom ULS-Viewer interpretiert und gefiltert werden kann. Nur relevant, wenn Logging in ULS aktiviert ist (einzustellen in Config.ps1)

Required? False
Position? 5
Default value powershell.exe (0x0E44)
Accept pipeline input? false
Accept wildcard characters? False

-thread <String>

Kann als zusätzlicher Parameter besetzt werden, da dieser Wert ebenfalls vom ULS-Viewer interpretiert und gefiltert werden kann. Nur relevant, wenn Logging in ULS aktiviert ist (einzustellen in Config.ps1)

Required? False
Position? 6
Default value 0x05BC
Accept pipeline input? false
Accept wildcard characters? False

-message <String>

Der eigentliche Log-Eintrag

Required? False
Position? 7
Default value  
Accept pipeline input? false
Accept wildcard characters? False

-InformationAction <ActionPreference>

Required? False
Position? named
Default value  
Accept pipeline input? false
Accept wildcard characters? False

-InformationVariable <String>

Required? False
Position? named
Default value  
Accept pipeline input? false
Accept wildcard characters? False

Inputs

Outputs

Notes

Der Einsatz dieses Cmdlets macht nur im Zusammenhang mit den SPE-Standard-Scripts Sinn, da ansonsten wichtige Umgebungsvariablen nicht gesetzt sind.

Examples

Beispiel 1 - Allgemeine Benutzung

PS C:\>  $scriptCorrId = $global:InitialCorrelationIDs.($Scriptname) #Wird in Config.ps1 definiert
 $global:CorrelationId = $scriptCorrId
Write-SPELogMessage -level "Critical" -area "Script" -category "Aborted" -message "Script has been aborted. Check Log(s)" -CorrelationId $scriptCorrId
